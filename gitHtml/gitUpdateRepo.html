updating and commiting the local repository with modified and changed files


The git command 'git status' will indicate any tracked and untracked files.
New files will be designated as untracked, files with changes will be marked as 'modified'.
Using 'git add filename filename etc,...' will pre-stage declared files to be commited to the repo.
Using 'git add .' will pre-stage all files with changes, to the repo. This saves abit of typing.
Then the 'git commit -m ""' command is issued to place these files into the local repo.
