git 'commit' command used to move files from staging area to a repository


The git command 'git commit' is the secondary and last step used to move staged files
to the repository (usually a local repository folder named .git)
Using 'git commit' will cause a message or window to appear, asking to user to enter
a comment or message that is relevant to the commiting of the staged files
Using 'git commit -m "comment"' will achieve the same thing except the message is directly
inserted using the -m command parameter followed by the message enclosed in double quotes
it is highly advised to add comments that are relevant to the files versioning information
